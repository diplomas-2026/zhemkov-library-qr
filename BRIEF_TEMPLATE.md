## 6) Технические ограничения
- Backend: Java 21, Spring Boot 3, Spring Security, JWT, PostgreSQL, Flyway, Docker Compose.
- Frontend: React, Vite, React Router.
- API через Docker, Web локально.

## 7) Seed и тестовые данные
- Заполни тестовыми данными все ключевые таблицы (не только пользователей).
- Seed должен быть идемпотентным.
- Данных должно хватать для полноценной проверки UI и E2E сценариев.

## 8) E2E и артефакты
- Обязателен Playwright E2E.
- Тесты должны:
  - логиниться тестовыми пользователями из `product-api/users.txt`
  - проверять ограничения доступа по выбранной модели
  - проверять ключевые бизнес-сценарии (create/edit/view/restrictions)
- Full-page скриншоты сохранять в `product-web/artifacts/screenshots/`.

## 9) GitHub и коммиты
- Работай в одном монорепозитории (корень: `product-api` + `product-web`).
- В начале создай репозиторий в организации `diplomas-2026` через GitHub CLI.
- Делай осмысленные коммиты по этапам и регулярно push.
- В конце все изменения должны быть запушены в GitHub.
